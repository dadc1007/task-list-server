# ¿Qué es mi producto y para qué sirve?

Mi producto es un servidor de lista de tareas que ayuda a organizar actividades diarias de forma simple y rápida.

En lugar de anotar pendientes en diferentes lugares, este servicio permite tener todas las tareas en un solo sistema, con la posibilidad de consultarlas, crear nuevas, actualizarlas y eliminarlas cuando ya no son necesarias.

Su objetivo principal es ahorrar tiempo y dar claridad sobre lo que está pendiente y lo que ya fue completado.

# ¿Cuáles son las funcionalidades más importantes y por qué los usuarios las usarían?

- Ver todas las tareas: permite tener una vista general del trabajo pendiente.
- Ver una tarea específica: facilita encontrar rápidamente el detalle de una actividad.
- Filtrar tareas completas o incompletas: ayuda a enfocarse en lo que falta por hacer o revisar avances.
- Crear tareas nuevas: permite registrar ideas o pendientes en el momento.
- Actualizar tareas existentes: es útil cuando cambian prioridades o descripciones.
- Eliminar tareas: mantiene la lista limpia y actualizada.

Los usuarios usarían estas funciones porque simplifican la organización diaria, reducen olvidos y mejoran el seguimiento de objetivos personales o de equipo.

# ¿Qué tecnologías use y por qué?

Para construir este producto utilicé Node.js y Express.

- Node.js permite crear aplicaciones rápidas y ligeras para manejar solicitudes en tiempo real.
- Express facilita construir el servidor con rutas claras para cada acción (consultar, crear, actualizar y eliminar tareas).

Elegí estas tecnologías porque son confiables, flexibles y permiten crecer el producto fácilmente en el futuro, por ejemplo agregando usuarios, base de datos o panel web.
